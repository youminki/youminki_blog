# 스타일 가이드 (Gemini Code Assist)

응답 언어: 한국어

이 파일은 Gemini Code Assist가 이 저장소에서 수행하는 코드 검토의 기본 규칙과 응답 방식을 지정합니다. 특히 모든 코드 리뷰 코멘트와 요약은 한국어로 작성되어야 합니다.

## 요약 형식

- 간단한 요약: 풀 리퀘스트/변경사항의 핵심 문제를 1-2문장으로 요약합니다.

- 권장 작업: 문제를 해결하기 위한 구체적이고 실행 가능한 권장사항을 번호 목록으로 제시합니다.

- 심각도 표기: 각 코멘트에 대해 LOW / MEDIUM / HIGH 중 하나의 심각도를 명시합니다.

- 코드 제안: 가능하면 변경 가능한 코드 스니펫(작은 패치)을 제공하여 바로 적용할 수 있도록 합니다.

## 코드 검토 중점 (카테고리)

1. 정확성 (Correctness)
   - 코드가 의도한 동작을 수행하는지 확인합니다.

   - 경계 사례, 널/undefined 처리, 타입 불일치, 비동기 로직의 오류 가능성(예: race condition) 등을 점검합니다.

2. 효율성 (Efficiency)
   - 불필요한 반복, 비효율적 데이터 구조 사용, 중복 계산, 메모리 과다 사용, 불필요한 네트워크/IO 호출 등을 지적합니다.

3. 유지관리성 (Maintainability)
   - 가독성, 함수/변수/파일 분리, 중복 코드, 주석과 문서 부족, 명명 규칙 위반, 매직 넘버 등을 지적합니다.

4. 보안 (Security)
   - 민감한 정보(시크릿) 하드코딩, 입력 검증 부족, 인젝션 취약점, 불충분한 권한 검증 등을 확인합니다.

5. 기타
   - 테스트(커버리지), 로깅과 에러 처리, 성능 및 확장성 고려사항, 운영 관찰성(모니터링/알림) 등을 검토합니다.

## 코멘트 작성 지침

- 모든 코멘트는 한국어로 작성합니다.

- 문제 설명 -> 왜 문제가 되는지(간단한 이유) -> 권장 해결책(또는 코드 예시)의 순서로 작성하세요.

- 가능한 경우 영향 범위(예: 파일명, 함수명, 라인 범위)를 명시합니다.

- 보안이나 민감 데이터 관련 이슈는 HIGH로 표기하고, 즉시 대응이 필요한 경우 별도 강조(예: **긴급: 시크릿 노출**)를 사용합니다.

- 스타일/형식 관련 코멘트는 친절한 문체로 제안하되, 프로젝트 컨벤션(예: TypeScript, React, Tailwind 등)을 존중하는 권장안으로 제시합니다.

## 예시 코멘트 (템플릿)

**심각도:** MEDIUM
**위치:** `src/components/SomeComponent.tsx` (라인 42-50)
**문제:** 이 컴포넌트는 props의 null/undefined 검사를 하지 않아 런타임 오류가 발생할 가능성이 있습니다.
**권장:** props를 비구조화할 때 기본값을 지정하거나, 타입을 엄격히 검사하세요. 예:

```ts
// 제안 코드
const { items = [] } = props;
```

## 금지/주의 사항

- 절대 시크릿(예: API 키, 비밀번호)을 코드에 남기지 않도록 지적하세요.
- 외부 네트워크 호출을 테스트 중에 노출시키는 코드가 있다면 모의(mock) 또는 환경변수 사용을 권장하세요.
